Tổng dãy | Editorial - Lời giải

Tag: Array, Math, Modulo, Basic

Statement - Đề bài

Cho một dãy AAA gồm nnn số nguyên dương a1,a2,a3,…,ana_1, a_2, a_3, \ldots, a_na​1​​,a​2​​,a​3​​,…,a​n​​ và một số nguyên dương kkk.

Yêu cầu: Tính tổng các phần tử trong dãy và đưa ra kết quả là phần dư của tổng vừa tìm cho kkk.

Input - Dữ liệu vào

  • Dòng đầu tiên gồm hai số nguyên dương n(1≤n≤1000)n \; (1 \le n \le 1000)n(1≤n≤1000) và k(1≤k≤1000)k \; (1 \le k \le 1000)k(1≤k≤1000).
  • Dòng thứ hai gồm nnn số nguyên a1,a2,a3,…,an(1≤ai≤1000)a_1, a_2, a_3, \ldots, a_n \; (1 \le a_i \le 1000)a​1​​,a​2​​,a​3​​,…,a​n​​(1≤a​i​​≤1000).

Output - Dữ liệu ra

  • Một số nguyên duy nhất là phần dư của phép chia giữa tổng các phần tử trong dãy AAA với số nguyên dương kkk.

Solution - Giải thuật

Naive solution - Giải thuật ngây thơ

Sử dụng vòng lặp for để tính tổng các phần tử trong dãy AAA sau đó lấy kết quả vừa tìm được chia lấy dư cho kkk.

Độ phức tạp thời gian: O(n)O(n)O(n).

C++:

#include <iostream>

using namespace std;

int n, k, sum = 0;

int main() {
    cin >> n >> k;
    for (int i = 1; i <= n; ++i) {
        int val;
        cin >> val;
        sum += val;
    }
    cout << sum % k << endl;
    return 0;
}

Optimized solution - Giải thuật tối ưu

Giả sử, dãy AAA có n=106n = 10^6n=10​6​​ giá trị a1,a2,a3,…,ana_1, a_2, a_3, \ldots, a_na​1​​,a​2​​,a​3​​,…,a​n​​ và ai=1018∀i∈[1,n]a_i = 10^{18} \forall i \in [1, n]a​i​​=10​18​​∀i∈[1,n] thì ta có tổng các phần tử trong dãy sẽ là: ∑i=11061018=106×1018=1024\sum\limits_{i = 1}^{10^6}10^{18} = 10^6 \times 10^{18} = 10^{24}​i=1​∑​10​6​​​​10​18​​=10​6​​×10​18​​=10​24​​ là một số rất lớn. Đối với một số ngôn ngữ lập trình không hỗ trợ xử lí số lớn như C++ thì sẽ không thể sử dụng các kiểu dữ liệu thông thường để lưu trữ giá trị này.

Do đó ta sẽ áp dụng tính chất (a+b)%c=(a%c+b%c)%c(a + b) \% c = (a \% c + b \% c) \% c(a+b)%c=(a%c+b%c)%c của đồng dư thức để xử lí bài toán. Cụ thể như sau:

  • Ta sử dụng vòng lặp for để duyệt qua từng phần tử trong dãy, sử dụng một biến sum để chứa giá trị tổng, cũng như là kết quả bài toán. Ta sẽ lần lượt cộng dồn a_i vào sum và công thức cộng dồn như sau: sum=sum+a[i]sum = sum + a[i]sum=sum+a[i].
  • Xét vế phải của công thức cộng dồn, lúc này, ta áp dụng tính chất vừa nêu của đồng dư thức ta được công thức sau: sum=(sum%k+a[i]%k)%ksum = (sum \% k + a[i] \% k) \% ksum=(sum%k+a[i]%k)%k. Khi được cộng dồn liên tục ta sẽ được kết quả cần tìm.

Ở lời giải này ta không tập trung vào việc tối ưu hóa thời gian tính tổng mà tập trung vào việc tối ưu hóa bộ nhớ và các tính. Vì số dư không bao giờ lớn hơn số chia nên kết quả bài toán luôn nằm trong khoảng [0,k−1][0, k - 1][0,k−1]. Do đó việc xử lí số lớn như giải thuật ngây thơ là không cần thiết.

Độ phức tạp thời gian: O(n)O(n)O(n).

C++:

#include <iostream>

using namespace std;

int n, k, sum = 0;

int main() {
    cin >> n >> k;
    for (int i = 1; i <= n; ++i) {
        int val;
        cin >> val;
        sum = (sum % k + val % k) % k;
    }
    cout << sum << endl;
    return 0;
}
