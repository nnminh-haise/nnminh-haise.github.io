          Modulo & Ước chung lớn nhất

          Gặp lại những phép toán mà ta đã bỏ qua khi còn ngồi trên ghế nhà trường.

          Modulo - Đồng dư thức

          Phép đồng dư thức là phép tính cho bạn số dư của số nguyên này cho số nguyên khác. Kí hiệu thường dùng cho phép đồng dư thức là %\%%.

          Ví dụ: ta có: 5÷2=25 \div 2 = 25÷2=2 dư 111, 5%2=15 \% 2 = 15%2=1.

          Một số tính chất của đồng dư thức

          • (a+b)%c=(a%c+b%c)%c(a + b) \% c = (a \% c + b \% c) \% c(a+b)%c=(a%c+b%c)%c.
          • (a⋅b)%c=((a%c)⋅(b%c))%c(a \cdot b) \% c = ((a \% c) \cdot (b \% c)) \% c(a⋅b)%c=((a%c)⋅(b%c))%c.

          Ví dụ: Với a=5,b=3,c=2a = 5, b = 3, c = 2a=5,b=3,c=2, ta có:

          • (5+3)%2=(5%2+3%2)%2=(1+1)%2=2%2=0(5 + 3) \% 2 = (5 \% 2 + 3 \% 2) \% 2 = (1 + 1) \% 2 = 2 \% 2 = 0(5+3)%2=(5%2+3%2)%2=(1+1)%2=2%2=0.
          • (5⋅3)%2=((5%2)⋅(3%2))%2=(1⋅1)%2=1%2=1(5 \cdot 3) \% 2 = ((5 \% 2) \cdot (3 \% 2)) \% 2 = (1 \cdot 1) \% 2 = 1 \% 2 = 1(5⋅3)%2=((5%2)⋅(3%2))%2=(1⋅1)%2=1%2=1.

          Ước chung lớn nhất

          Ước chung lớn nhất (hay còn gọi là Greatest Common Divisor - GCD) của hai hay nhiều số nguyên là số nguyên lớn nhất mà là ước chung của các số đó.

          Ví dụ: Ta có ước chung lớn nhất của 666 và 101010 là 222 vì 222 là số nguyên lớn nhất là ước chung của 666 và 101010.

          Thuật toán xác định ước chung lớn nhất

          Thuật toán ngây thơ

          Ta sẽ duyệt từ min(a,b)\min(a, b)min(a,b) về đến 111, nếu số đang xét hiện tại là ước của aaa và bbb thì số đó chính là gcd(a,b)\gcd(a, b)gcd(a,b)

          C++:

          int gcd(int a, int b) {
	for (int i = min(a, b); i >= 1; --i) {
		if (a % i == 0 && b % i == 0) {
			return i;
		}
	}
    // Chương trình sẽ không bao giờ chạy đến điểm này.
}

          Thuật toán Euclid

          Ban đầu thuật toán được mô tả như sau: với hai số nguyên aaa và bbb, ta sẽ lần lượt lấy số lớn hơn trừ cho số nhỏ hơn, cho đến khi một trong hai số bằng 000. Khi đó, số còn lại chính là ước chung lớn nhất của cả hai.

          Ví dụ: Với a=16,b=6a = 16, b = 6a=16,b=6 ta có:

          • Lần 111: ta lấy a−ba - ba−b, ta được a=10,b=6a = 10, b = 6a=10,b=6.
          • Lần 222: ta lấy a−ba - ba−b, ta được a=4,b=6a = 4, b = 6a=4,b=6.
          • Lần 333: ta lấy b−ab - ab−a, ta được a=4,b=2a = 4, b = 2a=4,b=2.
          • Lần 444: ta lấy a−ba - ba−b, ta được a=2,b=2a = 2, b = 2a=2,b=2.
          • Lần 555: ta lấy a−ba - ba−b, ta được a=0,b=2a = 0, b = 2a=0,b=2.

          Vì a=0a = 0a=0 nên ước chung lớn nhất của 161616 và 666 là 222.

          C++:

          int gcdEuclid(int a, int b) {
	while (a != 0 && b != 0) {
		if (a > b) {
			a -= b;
		}
		else {
			b -= a;
		}
	}

	if (a != 0) {
		return a;
	}

	return b;
}

          Mặt khác ta nhận xét sau: Với ba số nguyên a,b,ca, b, ca,b,c thỏa a−b=ca - b = ca−b=c ta có ccc chia hết cho gcd(a,b)\gcd(a, b)gcd(a,b).

          Chứng minh:

          Theo giả thuyết ta có: a−b=ca - b = ca−b=c. Gọi p=gcd(a,b)p = \gcd(a, b)p=gcd(a,b), ta có:

          {a=p×xb=p×y⇒a−b=p×x−p×y=p×(x−y)=c→c∣gcd(a,b)\begin{cases} a = p \times x \\ b = p \times y \end{cases} \Rightarrow a - b = p \times x - p \times y = p \times (x - y) = c \rightarrow c \mid \gcd(a, b) {​a=p×x​b=p×y​​⇒a−b=p×x−p×y=p×(x−y)=c→c∣gcd(a,b)

          Một cách tương tự, ta cũng có thể chứng minh được Với ba số nguyên a,b,ca, b, ca,b,c thỏa a−b=ca - b = ca−b=c ta có aaa chia hết cho gcd(b,c)\gcd(b, c)gcd(b,c).

          Từ hai nhận xét trên, ta rút ra được:

          {gcd(a,b)≤gcd(b,c)(b,c∣gcd(a,b))gcd(b,c)≤gcd(a,b)(a,b∣gcd(b,c))⇒gcd(a,b)=gcd(b,c)⇔gcd(a,b)=gcd(b,a−b)\begin{cases} \gcd(a, b) \le \gcd(b, c) \; (b, c \mid \gcd(a, b)) \\ \gcd(b, c) \le \gcd(a, b) \; (a, b \mid \gcd(b, c)) \end{cases} \Rightarrow \gcd(a, b) = \gcd(b, c) \Leftrightarrow \gcd(a, b) = \gcd(b, a - b) {​gcd(a,b)≤gcd(b,c)(b,c∣gcd(a,b))​gcd(b,c)≤gcd(a,b)(a,b∣gcd(b,c))​​⇒gcd(a,b)=gcd(b,c)⇔gcd(a,b)=gcd(b,a−b)

          Ta đã chứng minh được gcd(a,b)=gcd(b,a−b)\gcd(a, b) = \gcd(b, a - b)gcd(a,b)=gcd(b,a−b). Song, thứ tự phát biểu không quan trọng. Ta cũng có thể phát biểu: gcd(a,b)=gcd(a−b,b)\gcd(a, b) = \gcd(a - b, b)gcd(a,b)=gcd(a−b,b). Và khi ta tiếp tục khai triển theo công thức trên, ta được:

          gcd(a,b)=gcd(a−b,b)=gcd(a−2b,b)=gcd(a−3b,b)=⋯=gcd(a−qb,b)\gcd(a, b) = \gcd(a - b, b) = \gcd(a - 2b, b) = \gcd(a - 3b, b) = \cdots = \gcd(a - qb, b) gcd(a,b)=gcd(a−b,b)=gcd(a−2b,b)=gcd(a−3b,b)=⋯=gcd(a−qb,b)

          Mặt khác: a=qb+ra = qb + ra=qb+r với r=a%br = a \% br=a%b. Nên: a−qb=r⇒gcd(a,b)=gcd(r,b)=gcd(b,r)a - qb = r \Rightarrow \gcd(a, b) = \gcd(r, b) = \gcd(b, r)a−qb=r⇒gcd(a,b)=gcd(r,b)=gcd(b,r).

          Bội chung nhỏ nhất

          Bội chung nhỏ nhất của hai hay nhiều số (Least Common Multiple hay LCM) là số nguyên nhỏ nhất sao cho nó là bội của các số đó.

          Ta có thể xác định bội chung nhỏ nhất của hai số bằng cách lấy tích hai số đó và chia cho ước chung lớn nhất của nó:

          lcm(a,b)=a×bgcd(a,b)\text{lcm}(a, b) = \dfrac{a \times b}{\gcd(a, b)} lcm(a,b)=​gcd(a,b)​​a×b​​

          Do vậy ta có thể thiết kế một thuật toán tìm bội chung nhỏ nhất như sau:

          C++:

          int lcm(int a, int b) {
	return a * b / gcdEuclid(a, b);
}
